# API Migration Map: MonolithicFEMVLFS -> HydroElasticFEM

| Concern | Old (MonolithicFEMVLFS) | New (HydroElasticFEM) | Status | Notes |
|---|---|---|---|---|
| 1D beam model, frequency domain | `run_periodic_beam(::Periodic_Beam_params)` and `run_Khabakhpasheva_freq_domain(::Khabakhpasheva_freq_domain_params)` | `Physics.EulerBernoulliBeam` + `Simulation.FreqDomainConfig` + `Simulation.build_problem` + `Simulation.simulate` | CHANGED | Old code builds `AffineFEOperator` directly in script modules; new API is entity/context driven. |
| 1D beam model, time domain | `run_periodic_beam(::Periodic_Beam_params)` and `run_Khabakhpasheva_time_domain(::Khabakhpasheva_time_domain_params)` with `TransientConstantFEOperator` | `Physics.EulerBernoulliBeam` + `Simulation.TimeDomainConfig` + `Simulation.TimeConfig` + `Simulation.build_problem` + `Simulation.simulate(problem, tconfig)` | CHANGED | Old code used direct transient operators and script-level time setup. |
| 2D plate model, frequency domain | `run_Yago_freq_domain(::Yago_freq_domain_params)` / `run_MultiGeo_freq_domain(::MultiGeo_freq_domain_params)` | `Physics.KirchhoffLovePlate` + `Simulation.FreqDomainConfig` + `Simulation.build_problem` + `Simulation.simulate` | CHANGED | Plate constitutive tensor now available via `build_kl_tensor`/`build_KL_tensor`; orchestrator handles FE assembly. |
| Elastic joint boundary term | Explicit weak-form term in `src/Khabakhpasheva_freq_domain.jl` / `src/Khabakhpasheva_time_domain.jl` using `kᵣ` on `dΛj` | `Physics.JointRotationalSpring` attached via `EulerBernoulliBeam(joints=[...])` | CHANGED | Functionality exists but moved from script-level weak-form code into beam entity configuration. |
| Variable bathymetry (spatially varying H(x)) | Legacy scripts rely on custom geometry partitioning and hand-built domains; no stable exported API symbol dedicated to `H(x)` | No direct public `TankDomain` argument for bathymetry function; current public path is mesh-based domain setup | MISSING | Needed for paper section 5.4 as first-class user-facing setup (not ad hoc mesh pre-processing). |
| Linear system solve | `solve(op)` / `Gridap.solve(op)` and transient runs with `LUSolver()` in scripts | `Simulation.simulate(problem)` uses configured solver (`LUSolver` default) | FOUND | Same underlying solver concept; now wrapped by simulation orchestrator. |
| Time-domain time integrator | `Newmark(ls, dt, gamma_t, beta_t)` in script modules | `GeneralizedAlpha2(ls, tconfig.Δt, tconfig.ρ∞)` inside `Simulation.simulate(problem, tconfig)` | CHANGED | Integrator choice/parameters moved to `TimeConfig` and simulation module internals. |
| VTK export | `writevtk(...)`, `createvtk(...)`, `paraview_collection(...)` in scripts | Same Gridap/WriteVTK calls from user scripts/examples after solve | FOUND | No HydroElasticFEM wrapper required; VTK export remains script-level I/O. |
| Field extraction: deflection w | Direct tuple destructuring from solver output, e.g. `(ϕₕ, κₕ, ηₕ)` then evaluate at probes | `SimResult.solution` + `SimResult.fmap` field mapping, then evaluate FE functions at points | CHANGED | Data container changed (`SimResult`), but FE function probing pattern remains. |
| Field extraction: free-surface phi | Direct tuple access (`ϕₕ`, `κₕ`, `ηₕ`) and probe evaluation | `SimResult.solution` access with symbol-index map (`fmap`) | CHANGED | Extraction is no longer implicit positional API from script-local operator construction. |
